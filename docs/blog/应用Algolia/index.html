在 [Docusaurus] 中应用 Algolia

从零开始 Algolia​

暂略，可先查阅已有资源

爬取文档数据​

1. 配置爬虫所需环境和config​

  1. 在 Docusaurus 个人搭建的项目中，根目录新建 .env 文件，并根据要求输入Application ID 和 API Key。具体可在 Algolia 首页 API Keys 中进行查看。

    .env
    APPLICATION_ID=【你的Algolia Application ID】
    API_KEY=【你的Algolia API Key】

  2. 还是在项目的根目录中，新建文件 config.json，并 cv 下方的代码。

    有三个必须作更改的地方，分别是：

    • index_name：你的 Algolia index 名称
    • start_urls：你的 Docusaurus 网址
    • sitemap_urls：你的 Docusaurus 网址的 sitemap.xml
    config.json
    {
      "index_name": "ydsnotes",
      "start_urls": ["https://ydsuper.github.io/snotes/"],
      "sitemap_urls": ["https://ydsuper.github.io/snotes/sitemap.xml"],
      "stop_urls": ["/search"], 
      "selectors": {
        "lvl0": {
          "selector": "(//ul[contains(@class,'menu__list')]//a[contains(@class, 'menu__link menu__link--sublist menu__link--active')]/text() | //nav[contains(@class, 'navbar')]//a[contains(@class, 'navbar__link--active')]/text())[last()]",
          "type": "xpath",
          "global": true,
          "default_value": "Documentation"
        },
        "lvl1": "header h1, article h1",
        "lvl2": "article h2",
        "lvl3": "article h3",
        "lvl4": "article h4",
        "lvl5": "article h5, article td:first-child",
        "lvl6": "article h6",
        "text": "article p, article li, article td:last-child"
      },
      "custom_settings": {
        "attributesForFaceting": [
          "type",
          "lang",
          "language",
          "version",
          "docusaurus_tag"
        ],
        "attributesToRetrieve": [
          "hierarchy",
          "content",
          "anchor",
          "url",
          "url_without_anchor",
          "type"
        ],
        "attributesToHighlight": ["hierarchy", "content"],
        "attributesToSnippet": ["content:10"],
        "camelCaseAttributes": ["hierarchy", "content"],
        "searchableAttributes": [
          "unordered(hierarchy.lvl0)",
          "unordered(hierarchy.lvl1)",
          "unordered(hierarchy.lvl2)",
          "unordered(hierarchy.lvl3)",
          "unordered(hierarchy.lvl4)",
          "unordered(hierarchy.lvl5)",
          "unordered(hierarchy.lvl6)",
          "content"
        ],
        "distinct": true,
        "attributeForDistinct": "url",
        "customRanking": [
          "desc(weight.pageRank)",
          "desc(weight.level)",
          "asc(weight.position)"
        ],
        "ranking": [
          "words",
          "filters",
          "typo",
          "attribute",
          "proximity",
          "exact",
          "custom"
        ],
        "highlightPreTag": "<span class='algolia-docsearch-suggestion--highlight'>",
        "highlightPostTag": "</span>",
        "minWordSizefor1Typo": 3,
        "minWordSizefor2Typos": 7,
        "allowTyposOnNumericTokens": false,
        "minProximity": 1,
        "ignorePlurals": true,
        "advancedSyntax": true,
        "attributeCriteriaComputedByMinProximity": true,
        "removeWordsIfNoResults": "allOptional",
        "separatorsToIndex": "_",
        "synonyms": [
          ["js", "javascript"],
          ["ts", "typescript"]
        ]
      }
    }

2. 安装爬虫所需工具​

此步比较关键⭐，由于我使用的是 Windows 系统，此步略微繁琐，跟着下方步骤来，步步大道。

  1. 安装 Docker Desktop，爬取时需要使用 Docker Daemon。

    ⚠值得注意的是：可能因为软件版本的问题，无法安装，可手动选择其他版本下载后尝试安装

  2. 安装 chocolatery，因为下一步需要通过 chocolatery 安装 jq。将 CMD 以管理员身份运行，并执行下方指令。

    @"%SystemRoot%\System32\WindowsPowerShell\v1.0\powershell.exe" -NoProfile -InputFormat None -ExecutionPolicy Bypass -Command "iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))" && SET "PATH=%PATH%;%ALLUSERSPROFILE%\chocolatey\bin"

  3. 使用 chocolatery 安装 jq。直接执行下方指令进行安装。

    chocolatey install jq

3. 实践爬取并同步到应用索引​

当准备工作都完成后，那么现在可以实践爬取数据的过程。现在使用 Git Bash 打开我们的 Docusaurus 项目，如果你是 Mac 请输入以下指令：

docker run -it --env-file=.env -e "CONFIG=$(cat ./config.json | jq -r tostring)" algolia/docsearch-scraper

Windows 系统需要使用 winpty，使用以下指令：

winpty docker run -it --env-file=.env -e "CONFIG=$(cat ./config.json | jq -r tostring)" algolia/docsearch-scraper

执行完后，稍等片刻......

执行成功后，即可在 Algolia 平台的该应用的后台，Search 页面中，查看网站的爬取数据是否同步到索引(index)下

搜索功能

index已有站点的搜索数据，那么现在就可以运行项目，测试功能是否实现，搜索成功~~ 即 Algolia 成功落地啦🎉
